# Gate raw Herdr tasking inside Pi

The package will apply one Herdr command classifier to both built-in `bash` tool calls and `user_bash` (`!` / `!!`) events. Recognized scoped metadata inspection and reads of this Pi's own pane remain available, with output framed as untrusted data. Foreign `pane read` / `agent read` are redirected to the explicitly user-authorized `herdr_agent_output_inspect` tool, and `api snapshot` is denied because it would bypass current-Workspace-Scope metadata rules. Raw tasking, Agent creation, foreign-pane control, and blocking waits are also denied and redirected to `herdr_dispatch_propose` or `/herdr-dispatch`. This is necessary because the official globally installed Herdr skill teaches the Origin Pi itself to use raw `herdr pane`, `herdr agent`, and `herdr wait` commands, which would otherwise bypass proposal immutability, Target Occupancy, Worktree Write Leases, inspection authorization, monitoring, and the result protocol. Dispatch tools also carry prompt guidelines requiring all inter-Agent tasking to use the typed dispatch interface. The gate is best-effort rather than a shell sandbox: external shells, obfuscated commands, direct socket clients, unknown custom tools, and indirect execution paths the classifier cannot resolve (for example `find -exec`) remain outside its guarantee.
