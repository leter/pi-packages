# Gate raw Herdr tasking inside Pi

The package will apply one Herdr command classifier to both built-in `bash` tool calls and `user_bash` (`!` / `!!`) events. Recognized read-only inspection remains available, with output framed as untrusted data, while raw tasking, Agent creation, foreign-pane control, and blocking waits are denied and redirected to `herdr_dispatch_propose` or `/herdr-dispatch`. This is necessary because the official globally installed Herdr skill teaches the Origin Pi itself to use raw `herdr pane`, `herdr agent`, and `herdr wait` commands, which would otherwise bypass confirmation, Target Occupancy, Worktree Write Leases, and the result protocol. Dispatch tools also carry prompt guidelines requiring all inter-Agent tasking to use the typed dispatch interface. The gate is best-effort rather than a shell sandbox: external shells, obfuscated commands, direct socket clients, and unknown custom tools remain outside its guarantee.
