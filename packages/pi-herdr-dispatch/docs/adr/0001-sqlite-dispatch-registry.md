# Use SQLite for the global Dispatch Registry

The package will persist dispatches, target occupancy, worktree write leases, results, and audit events in SQLite using WAL mode, transactions, and unique constraints. A single JSON file was rejected because multiple Pi processes need atomic reservation checks, while Herdr metadata is display-oriented rather than a durable transactional source of truth. SQLite does not make Herdr delivery transactional, so a durable `delivering` intent and explicit delivery-verification protocol handle that external-side-effect gap. Registry corruption, migration failure, or structurally unexpected SQL errors fail closed; the package must not create an empty replacement or fall back to memory because doing so would silently discard unsettled reservations. Exhausting `busy_timeout` with `SQLITE_BUSY`/`SQLITE_LOCKED` fails the current operation but does not permanently trip the process-local mutation fuse: ordinary multi-process contention is transient, the transaction has not begun, and a later operation may safely retry. Structural errors still disable subsequent mutations for that process while safe reads remain available.
