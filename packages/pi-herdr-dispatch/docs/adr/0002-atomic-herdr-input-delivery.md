# Deliver dispatches through atomic Herdr pane input

Dispatch messages will use Herdr's socket-level `pane.send_input` method to send complete text plus Enter in one request. Immediately beforehand, the adapter re-resolves the confirmed terminal identity and verifies the returned pane on the same connection; it also observes pane close/move events during delivery and verifies the unique dispatch echo afterward. This removes the split text/Enter race but cannot eliminate the residual race between final route validation and server-side input handling, so ambiguous delivery remains `delivery-unverified` and is never automatically resent. Protocol incompatibility fails closed rather than falling back to split delivery.
