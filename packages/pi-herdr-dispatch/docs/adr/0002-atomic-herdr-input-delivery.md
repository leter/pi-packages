# Deliver dispatches through atomic Herdr pane input

> Superseded in part by [ADR 0006](./0006-stage-multiline-input-before-submit.md): real Claude Code multiline input requires staged text and a separately submitted Enter.

Dispatch messages will use Herdr's socket-level `pane.send_input` method to send complete text plus Enter in one request. Immediately beforehand, the adapter re-resolves the confirmed terminal identity and verifies the returned pane through tightly adjacent unary requests. Herdr 0.7.3 closes a unary connection after its first response, while `events.subscribe` owns a separate long-lived stream, so each unary operation uses a fresh socket ([compatibility spike](../SPIKE-RESULTS.md#7-unary-and-subscription-socket-lifecycles)). The adapter observes pane close/move events during delivery, checks route invalidation immediately before writing the send request, and verifies the unique dispatch echo afterward. This removes the split text/Enter race but cannot eliminate the residual race between final route validation and server-side input handling. Herdr's official agent skill guarantees that “Closed resources don't reuse IDs,” so closing a pane cannot retarget its old ID to a later resource; moving a pane still assigns a new public pane ID and may make the route stale ([Herdr agent skill](https://herdr.dev/docs/agent-skill/)). Ambiguous delivery remains `delivery-unverified` and is never automatically resent. Protocol incompatibility and unary transport errors fail closed rather than falling back to split delivery.
